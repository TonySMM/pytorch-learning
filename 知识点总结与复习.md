# PyTorch 深度学习全课程知识点总结与复习核心

这份文档为你总结了整个 PyTorch 学习过程中的核心知识点。内容从基础构建过渡到高级模型，**重点强化了后半部分（CNN、RNN及数据处理）的复习与避坑指南**。

---

## 第一部分：PyTorch 核心基础（02-07讲）

### 1. 深度学习的“标准流程”三步曲
不管是多复杂的模型，在 PyTorch 中训练的基本框架是不变的：
1. **前向传播 (Forward)**：`y_pred = model(x)`
   - 将输入数据喂给模型，得到预测值。
2. **计算损失 (Compute Loss)**：`loss = criterion(y_pred, y)`
   - 根据预测值和真实标签计算误差。
3. **反向传播与更新 (Backward & Update)**：
   - `optimizer.zero_grad()`：**关键**，清空历史梯度，防止梯度累积。
   - `loss.backward()`：由自动求导机制根据计算图反向计算所有可导参数的梯度。
   - `optimizer.step()`：优化器根据梯度更新权重。

### 2. 核心组件
- **`nn.Module`**：所有自定义神经网络模块的基类。至少需要重写 `__init__()`（定义层）和 `forward()`（定义前向传播的计算过程）。
- **`torch.optim`**：优化器家族。如 `optim.SGD`，`optim.Adam`。理解 `momentum` 动量的作用：累积历史梯度方向，加速收敛并减少震荡。
- **Sigmoid vs Softmax**：
  - 二分类用 `nn.BCELoss`，网络末端配合 `Sigmoid`（或直接使用 `nn.BCEWithLogitsLoss`）。
  - 多分类用 `nn.CrossEntropyLoss`，网络末端**不需要**加 Softmax（因为损失函数内部已经包含了 Softmax + Log + NLLLoss 操作）。

## 第二部分：数据处理与加载（08讲：重中之重）

在实际工程中，写数据处理代码的时间往往比写模型本身还要长。

### 1. `Dataset` 和 `DataLoader`
为了支持 mini-batch 训练和并行加载，我们必须掌握这两个类。

**自定义 Dataset 必须重写三个魔法方法：**
- `__init__(self, ...)`：读取数据本身或者包含所有数据路径的列表。
- `__getitem__(self, index)`：根据索引 `index` 提取出**一条**训练数据及其标签，通常在这里转化成 Tensor。
- `__len__(self)`：返回数据集的总样本数，决定了 DataLoader 多久算一个 epoch。

**DataLoader：**
负责将 Dataset 的数据打包成 Batch。常用的参数：`batch_size`, `shuffle` (打乱，一般训练集为 True，测试集为 False), `num_workers` (多进程加载数据，Windows 下可能报错，常设为 0)。

## 第三部分：卷积神经网络 CNN（10-11讲）

相比于全连接层（Linear），CNN 更适合处理图像，因为它利用了**局部相关性**和**权重共享**。

### 1. CNN 基础参数
对于 `nn.Conv2d(in_channels, out_channels, kernel_size, stride=1, padding=0)`：
- **`in_channels`**：输入通道数（如 RGB 图像是 3）。
- **`out_channels`**：输出通道数（即**卷积核的数量**），决定了提取了多少种特征。
- **`kernel_size`**：卷积核大小（如 3x3，5x5）。
- **`stride`**：步长，决定特征图迷你的比例。
- **`padding`**：填充，周围补0以保留边缘信息，并控制输出大小。

### 2. 特征图（Feature Map）维度变化计算
假定输入为 $W \times W$，卷积核为 $K$，步长 $S$，填充 $P$，则输出边长为：
$$ Output = \lfloor \frac{W - K + 2P}{S} \rfloor + 1 $$
*(注意：到了池化层 `MaxPool2d`，维度计算公式也是类似的。)*

### 3. 高级 CNN 结构
- **1x1 卷积**：主要用来改变**通道数**（升维或降维），减少参数量，并增加非线性表达能力。
- **Inception Module (GoogleNet 思想)**：让网络自己决定用几乘几的卷积核。通过拼接不同视野的卷积结果（`torch.cat([out1, out2, out3], dim=1)`），融合多尺度特征。
- **Residual Block (惨差网络 ResNet 思想)**：解决梯度消失问题。核心公式 `H(x) = F(x) + x`。把浅层的输入跨层直接加到深层输出上，使得网络可以轻易做深。

## 第四部分：循环神经网络 RNN（12-13讲：最难啃的硬骨头）

RNN 是处理时间序列（如文本序列，时间序列预测）的霸主，它的核心是**记忆体（Hidden State）**。

### 1. 核心维度概念
初学 RNN 最容易被维度搞晕。在 PyTorch 中：
- `input_size`：每一帧/词向量的特征维度。
- `hidden_size`：记忆体隐藏层的维度（决定了模型的“脑容量”）。
- `seq_len`：序列的长度。
- `num_layers`：RNN层叠层数。

**输入 Tensor 的形状：**
默认情况下，PyTorch 的 RNN 需要的输入维度是 `(seq_len, batch_size, input_size)`。
*为啥 `seq_len` 在前面？因为它是一个个时间步循环执行的，这样内存排列更有利于底层 C++ 函数遍历加速。*
如果你更习惯第一维是 batch，需要在定义 RNN 时指定 `batch_first=True`，这样输入就必须是 `(batch_size, seq_len, input_size)`。

### 2. 多种序列模型
- **RNNCell / GRUCell / LSTMCell**：这只是“基本运算单元”，每跑一步只能处理**一个时间步**。要处理一整个序列，你需要自己手写 `for` 循环。
- **RNN / GRU / LSTM**：这才是高度封装的“层”。你可以一次性把所有序列喂给它，底层自动完成了 `for` 循环（C++加速层），直接吐出所有时间步的输出以及最后一个时间步的隐藏状态。

### 3. 变长序列的高级技巧 (重点与踩坑点)
在处理文本等变长数据时，我们通常需要填充（Padding）让同一批次的句子一样长。但这会导致 RNN 去计算毫无意义的 padding 0，浪费算力且干扰状态。
**解决办法：`pack_padded_sequence` (打包填充序列)**

**使用步骤与坑：**
1. **降序排序**：对当前批次按照序列的真实长度从大到小排序。
2. **压扁**：使用 `pack_padded_sequence(inputs, seq_lengths)`。
   - ⚠️ **天坑警告**：PyTorch 底层 C++ / CUDA 实现要求这里的 `seq_lengths` **必须是 CPU 上的 1D Int64 张量**。绝对不要把 `seq_lengths` 给 `.to(device)` 或 `.cuda()`，否则会报 RuntimeError！
3. **输入 RNN**：将打包后的对象作为输入直接传给 GRU/LSTM。
4. **解包 (可选)**：如果后续需要完整的输出形状来进行计算（比如用 Transformer 注意力），就用 `pad_packed_sequence()` 把结果转换回填充着0的正常 Tensor 张量。

## 给你的学习建议

1. **维度是深度学习的脉门**：在看代码（尤其是复杂的网络结构部分，如 `forward` 函数内）的时候，用注释把每个变量经历过运算后的 `(Batch, Channel, Height, Width)` 或 `(Seq_len, Batch, Hidden)` 等形状写下来，这是不至于看晕的最有效方法！
2. **把模型当成黑盒**：在刚接触 `GRU` 或 `ResNet` 等模型时，先不要去管内部繁杂的公式。只要明白它是“干嘛的”、“输入是什么维度”、“输出是什么维度”，先会组装起来跑通，再去深究。
3. **Debug 大招**：如果是报维度不匹配的错，使用 `print(tensor.shape)` 是万恶之源也是解题之匙。
